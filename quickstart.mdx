# Quick Start Guide

Get CryptoLay up and running in under 5 minutes. This guide will walk you through setting up the development environment and making your first API call.

## Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js 18+** - [Download here](https://nodejs.org/)
- **Docker & Docker Compose** - [Download here](https://www.docker.com/products/docker-desktop/)
- **PostgreSQL** - Included with Docker setup
- **AWS Account** - For KMS and Lambda deployment (optional for local development)

## Step 1: Clone the Repository

```bash
git clone https://github.com/TorresAndy/cryptolay.git
cd cryptolay
```

## Step 2: Start the Database

```bash
docker-compose up postgres -d
```

This starts a PostgreSQL database on port 5432.

## Step 3: Apply Database Schema

```bash
psql -h localhost -U postgres -d cryptolay -f database/schema.sql
```

If you don't have `psql` installed, you can use Docker:

```bash
docker exec -i cryptolay_postgres_1 psql -U postgres -d cryptolay < database/schema.sql
```

## Step 4: Start the Main API

```bash
cd main-api
npm install
npm run dev
```

The API will be available at `http://localhost:3000`.

## Step 5: Create Your First API Key

Before you can make API calls, you need an API key. For development, you can create a test key:

```bash
curl -X POST http://localhost:3000/api/customer/api-keys \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Test API Key",
    "permissions": ["encrypt", "decrypt"]
  }'
```

## Step 6: Create a Manifest

A manifest defines which fields should be encrypted in your data:

```bash
curl -X POST http://localhost:3000/api/customer/manifests \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{
    "name": "User Data Manifest",
    "fields": ["email", "phone", "ssn"],
    "description": "Encrypts sensitive user information"
  }'
```

## Step 7: Encrypt Your First Data

Now you can encrypt sensitive data:

```bash
curl -X POST http://localhost:3000/api/encrypt \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{
    "manifest_id": "YOUR_MANIFEST_ID",
    "data": {
      "email": "user@example.com",
      "phone": "+1234567890",
      "ssn": "123-45-6789"
    }
  }'
```

## Step 8: Decrypt Your Data

To decrypt the data, you'll need the encrypted response and metadata:

```bash
curl -X POST http://localhost:3000/api/decrypt \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{
    "manifest_id": "YOUR_MANIFEST_ID",
    "data": {
      "email": "ENCRYPTED_EMAIL_VALUE",
      "phone": "ENCRYPTED_PHONE_VALUE",
      "ssn": "ENCRYPTED_SSN_VALUE"
    },
    "meta": {
      "identifier": "REQUEST_UUID",
      "encrypted_key": "KMS_ENCRYPTED_AES_KEY",
      "key_fingerprint": "KMS_KEY_ID",
      "timestamp": "2024-01-01T00:00:00Z"
    }
  }'
```

## Environment Variables

Create a `.env` file in the `main-api` directory:

```env
DATABASE_URL=postgresql://postgres:password@localhost:5432/cryptolay
AWS_REGION=us-east-1
AWS_ACCESS_KEY_ID=your_key
AWS_SECRET_ACCESS_KEY=your_secret
NODE_ENV=development
```

For local development without AWS KMS, you can use the mock KMS implementation.

## Troubleshooting

### Database Connection Issues

If you can't connect to the database:

```bash
# Check if PostgreSQL is running
docker ps

# Restart the database
docker-compose restart postgres
```

### API Key Issues

Make sure your API key is valid and has the correct permissions:

```bash
# Check API key status
curl -X GET http://localhost:3000/api/customer/api-keys \
  -H "X-API-Key: YOUR_API_KEY"
```

### Port Conflicts

If ports are already in use, you can change them in the respective `package.json` files or stop the conflicting services.
