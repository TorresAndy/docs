# Quick Start Guide

Get CryptoLay up and running in under 5 minutes. This guide will walk you through setting up your account and making your first API call.

## Prerequisites

Before you begin, make sure you have:

- A web browser
- Basic knowledge of REST APIs
- Your application ready for integration

## Step 1: Create Your Account

1. Visit [cryptolay.com](https://cryptolay.com)
2. Click **Get Started** or **Sign Up**
3. Fill in your details and create your account
4. Verify your email address

## Step 2: Create Your First API Key

1. Log into your CryptoLay dashboard
2. Navigate to **API Keys** in the sidebar
3. Click **Create New API Key**
4. Fill in the details:
   - **Name**: A descriptive name (e.g., "Production API Key")
   - **Permissions**: Select "encrypt" and "decrypt"
5. Click **Create Key**
6. Copy your API key (you'll only see it once!)

## Step 3: Create a Manifest

A manifest defines which fields should be encrypted in your data:

1. Go to **Manifests** in your dashboard
2. Click **Create New Manifest**
3. Fill in the details:
   - **Name**: "User Data Manifest"
   - **Fields**: Add the field names you want to encrypt (e.g., "email", "phone", "ssn")
   - **Description**: "Encrypts sensitive user information"
4. Click **Create Manifest**

## Step 4: Encrypt Your First Data

Now you can encrypt sensitive data using your manifest:

```bash
curl -X POST https://api.cryptolay.com/api/encrypt \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{
    "manifest_id": "YOUR_MANIFEST_ID",
    "data": {
      "email": "user@example.com",
      "phone": "+1234567890",
      "ssn": "123-45-6789"
    }
  }'
```

## Step 5: Decrypt Your Data

To decrypt the data, you'll need the encrypted response and metadata:

```bash
curl -X POST https://api.cryptolay.com/api/decrypt \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{
    "manifest_id": "YOUR_MANIFEST_ID",
    "data": {
      "email": "ENCRYPTED_EMAIL_VALUE",
      "phone": "ENCRYPTED_PHONE_VALUE",
      "ssn": "ENCRYPTED_SSN_VALUE"
    },
    "meta": {
      "identifier": "REQUEST_UUID",
      "encrypted_key": "KMS_ENCRYPTED_AES_KEY",
      "key_fingerprint": "KMS_KEY_ID",
      "timestamp": "2024-01-01T00:00:00Z",
      "algorithm": "AES-256-GCM"
    }
  }'
```

## JavaScript Example

```javascript
const response = await fetch('https://api.cryptolay.com/api/encrypt', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-API-Key': 'your_api_key'
  },
  body: JSON.stringify({
    manifest_id: 'your_manifest_id',
    data: {
      email: 'user@example.com',
      phone: '+1234567890'
    }
  })
});

const result = await response.json();
console.log(result.encrypted_data);
```

## Python Example

```python
import requests

response = requests.post(
    'https://api.cryptolay.com/api/encrypt',
    headers={
        'Content-Type': 'application/json',
        'X-API-Key': 'your_api_key'
    },
    json={
        'manifest_id': 'your_manifest_id',
        'data': {
            'email': 'user@example.com',
            'phone': '+1234567890'
        }
    }
)

result = response.json()
print(result['encrypted_data'])
```

## Environment Variables

Store your API key securely using environment variables:

```bash
# Development
export CRYPTOLAY_API_KEY="your_api_key"

# Production
export CRYPTOLAY_API_KEY="your_production_api_key"
```

## Testing Your Setup

Test your integration with a simple encryption:

```bash
curl -X POST https://api.cryptolay.com/api/encrypt \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{
    "manifest_id": "YOUR_MANIFEST_ID",
    "data": {
      "email": "test@example.com"
    }
  }'
```

If successful, you should receive a response with encrypted data and metadata.

## Troubleshooting

### Invalid API Key

If you get a 401 error:
1. Check that your API key is correct
2. Verify the API key hasn't expired
3. Ensure the API key has the required permissions

### Invalid Manifest

If you get a 400 error:
1. Check that the manifest ID is correct
2. Verify the manifest is active
3. Ensure the data fields match the manifest

### Rate Limiting

If you get a 429 error:
1. Check your current usage limits
2. Implement exponential backoff
3. Consider upgrading your plan
